集成mybatis

mybatis-spring-boot-starter起步依赖提供了集成mybatis的简便方式

添加maven依赖

 1<dependency>
 2    <groupId>org.mybatis.spring.boot</groupId>
 3    <artifactId>mybatis-spring-boot-starter</artifactId>
 4    <version>2.2.0</version>
 5</dependency>
 6
 7<dependency>
 8    <groupId>mysql</groupId>
 9    <artifactId>mysql-connector-java</artifactId>
10</dependency>

配置

数据源相关配置信息

1spring:
2  datasource:
3    driver-class-name: com.mysql.cj.jdbc.Driver
4    url: jdbc:mysql://localhost:3306/test?useUnicode=true&characterEncoding=utf-8&useSSL=false
5    username: root
6    password: root

mybatis相关配置信息

1mybatis:
2  # config-location: classpath:mybatis-config.xml # mybatis全局配置文件路径
3  mapper-locations: classpath:com/lypzz/mapper/*.xml # mapper映射文件路径
4  type-aliases-package: com.lypzz.entity # 实体类支持使用别名(类名或类名首字母小写)，多个包之间用【,; \t\n】分开
5  configuration:
6    map-underscore-to-camel-case: true # 自动映射驼峰命名与下划线

使用mybatis

与直接使用mybatis相比，mybatis-spring-boot-starter 的简化了很多，无需手动加载配置文件，不用手动创建 SqlSessionFactory 以及 SqlSession 对象，只需要编写Mapper接口以及xml映射文件（或基于注解的配置）即可。

定义实体类

com.lypzz.entity.Employee

 1@Data
 2public class Employee
 3{
 4    private int number; // 员工编号
 5    private Date birthday; // 出生日期
 6    private String firstName;
 7    private String lastName;
 8    private String gender; // 性别
 9    private Date hireDay; // 雇佣日期
10}

定义Mapper接口

要想被Spring容器发现，需要在接口上使用注解@Mapper，或者在启动引导类上使用 @MapperScan 指定要扫描的包

com.lypzz.mapper.EmployeeMapper

1@Mapper
2public interface EmployeeMapper
3{
4    List<Employee> findAll();
5}

编写xml映射文件

注意：xml映射文件的路径要与对应的mapper接口包名对应，同时必须与配置中的mapper-locations相同

com/lypzz/mapper/EmployeeMapper.xml

 1<?xml version="1.0" encoding="UTF-8" ?>
 2<!DOCTYPE mapper
 3        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
 4        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
 5<mapper namespace="com.lypzz.mapper.EmployeeMapper">
 6    <resultMap id="employeeResultMap" type="employee">
 7        <result property="number" column="emp_no"/>
 8        <result property="birthday" column="birth_date"/>
 9        <result property="firstName" column="first_name"/>
10        <result property="lastName" column="last_name"/>
11        <result property="gender" column="gender"/>
12        <result property="hireDay" column="hire_date"/>
13    </resultMap>
14
15    <select id="findAll" resultMap="employeeResultMap">
16        select * from employees;
17    </select>
18</mapper>

执行crud

com.lypzz.controller.MyController

 1@RestController
 2public class MyController
 3{
 4    private final EmployeeMapper employeeMapper;
 5
 6    @Autowired
 7    public MyController(EmployeeMapper employeeMapper)
 8    {
 9        this.employeeMapper = employeeMapper;
10    }
11
12    @GetMapping("/employees")
13    public Object showAllEmployees()
14    {
15        List<Employee> employees = employeeMapper.findAll();
16        return employees;
17    }
18}

修改数据源

spring-boot jdbc默认使用 Hikari 作为数据源，如果要使用其它数据源，需要进行配置。

使用Druid数据源

官网文档

  • https://github.com/alibaba/druid/wiki/
  • https://hub.fastgit.org/alibaba/druid/tree/master/druid-spring-boot-starter

添加maven依赖

1<dependency>
2    <groupId>com.alibaba</groupId>
3    <artifactId>druid-spring-boot-starter</artifactId>
4    <version>1.2.6</version>
5</dependency>

在配置文件中设置数据源为 druid

1spring.datasource.type = com.alibaba.druid.pool.DruidDataSource

druid提供的配置项以 spring.datasource.druid 开头，具体见文档。

集成PageHelper插件

PageHelper插件文档：https://github.com/pagehelper/Mybatis-PageHelper/blob/master/wikis/zh/HowToUse.md

添加maven依赖

1<!-- https://mvnrepository.com/artifact/com.github.pagehelper/pagehelper-spring-boot-starter -->
2<dependency>
3    <groupId>com.github.pagehelper</groupId>
4    <artifactId>pagehelper-spring-boot-starter</artifactId>
5    <version>1.4.1</version>
6</dependency>

配置

1pagehelper:
2#  auto-dialect: true # 是否开启自动检测sql方言
3  helper-dialect: mysql # 设置sql方言
4  reasonable: true # 合理化分页参数，默认false
5  params: pageNum=pageNum;pageSize=pageSize;count=countSql;reasonable=reasonable;pageSizeZero=pageSizeZero
6  row-bounds-with-count: false

使用

只需要执行具体查询之前，使用 PageHelper.startPage() 方法即可

注意：PageHelper插件在分页时，会在mapper中的SQL语句后面进行拼接，所以，SQL语句不要加分号 ;

1int pageNum = 1;
2int pageSize = 3;
3String orderBy = "emp_no DESC";
4
5PageHelper.startPage(pageNum, pageSize,orderBy);
6List<Employee> employees = employeeMapper.findAll();

可以将查询结果包装到 PageInfo 对象中，以获取详细的分页信息

1PageInfo<Employee> page = new PageInfo<>(employees);
